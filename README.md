## Code & visualisation

(This has been generated by claude-ai. This describes an example of the how the structure of the project relates to the workflow.)

The Julia codebase is split into three layers:

- **`models/`** — equations of motion; import these everywhere
- **`analysis/`** — Poincaré sections, Lyapunov exponents, etc.
- **`scripts/explore.jl`** — interactive GLMakie dashboard with sliders for energy and initial conditions; plots update live
- **`notebooks/`** — Pluto reactive notebooks for structured exploration

Figures are saved to `figures/` and referenced in notes with relative paths. Data files are saved to `data/` and shared between code and notes.

To run the interactive dashboard:
```julia
julia --project=code code/scripts/explore.jl
```

To open the Pluto notebook:
```julia
julia --project=code -e 'using Pluto; Pluto.run()'
```

## Glossary workflow

(This has been gernerated by claude-ai to describe the workflow which I implemented with the help of claude-ai.)

`notes/glossary.json` is the single source of truth for all definitions and LaTeX symbols.
Never edit `glossary.qmd` or `styles/macros.html` directly — they are generated.

**Adding a term:**

```json
"phase space": {
  "def": "The space of all possible states of the system.",
  "date": "2026-05-27",
  "macro": "ps",
  "latex": "(x, y, p_x, p_y)"
}
```

- `def` — plain text definition, rendered in the glossary page
- `macro` — shorthand name for the LaTeX macro (omit or set `null` if not needed)
- `latex` — the LaTeX expression the macro expands to

**Regenerate after editing:**

```bash
bash build.sh
```

This runs `sync_glossary.sh` which writes `glossary.qmd` and `styles/macros.html`, then renders the full site.

**Using a macro in notes:**

```markdown
The Hamiltonian $\HH$ governs the dynamics in $\ps$.
```
